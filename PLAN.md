# План работ

Этот файл теперь ведётся только под сервисный вариант: один рабочий `Shadowsocks`-узел, текстовые конфиги и простой деплой.

## Цель

Собрать рабочий `Shadowsocks`-сервис без собственного клиента и backend:

1. один VPS;
2. один рабочий `ssserver`;
3. один или несколько текстовых `ss://` конфигов;
4. быстрый деплой и переиздание конфигов;
5. проверка реальной работы в сторонних клиентах.
